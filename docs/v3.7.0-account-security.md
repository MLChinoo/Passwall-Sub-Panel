# v3.7.0 «账号安全» 设计稿

本地认证的一整套账号安全能力，作为 **v3.7.0** 专题、跨多个 beta 按依赖顺序交付。
6 块功能落在同一套公共底座上，每块都**可配置开关**，默认全部**只针对本地账号**
（SSO 用户无本地密码，且 MFA 由 IdP 负责——PSP 不重复做 SSO 侧 2FA）。

> 约定：本文档中文；代码注释英文；严守「身份零隐式 fallback」（[[feedback_no_implicit_identity_fallback]]）；
> schema 改动直接加列/加表走 AutoMigrate、不写迁移脚本（[[project_no_migrations]]）；
> 严格 TDD（先红后绿，[[feedback_tdd]]）；关键路径真机验证（[[feedback_live_panel_testing]]）。

## 0. 范围与版本

| 项 | 决定 |
|---|---|
| 版本 | v3.7.0「账号安全」专题，跨多 beta，最后汇总正式版 |
| 适用范围 | **默认仅本地账号**（local bcrypt 登录）；SSO 不受影响、不加 2FA |
| 全可配 | 每块独立开关，管理员在「设置 → 登录安全」统一配置 |

## 1. 公共底座（先建，后 5 块复用）

1. **验证码 + 限流网关** —— 所有新公开端点（forgot / reset / register / passkey-begin）
   一律挂在登录限流器 + 验证码后面。**这是先做验证码/锁定的根本原因**：它是后面一切的闸门。
2. **`auth_tokens` 单表** —— 一次性、哈希存储、带 TTL，`purpose ∈ {password_reset, email_verify}`，
   同时服务「找回密码」与「注册邮箱验证」。字段：`id, user_id(可空), purpose, token_hash, code_hash,
   email, expires_at, used_at, created_at`。同时支持**链接**（长随机 token）与 **OTP**（6 位码，单独 `code_hash`）。AutoMigrate。
3. **登录中间挑战令牌** —— 密码+验证码过了但需 2FA 时，签发短时 `2fa_pending` JWT（非完整 access token），
   `/auth/2fa/verify` 用它换真 token。无状态、无表。
4. **公开认证配置扩展** —— `/auth/methods`（`Methods()`）增暴露：注册/找回/passkey 是否开、验证码 provider+site_key+是否要求。
   前端据此渲染登录/注册/找回入口。
5. **邮件模板复用** —— 复用现有 MailReminder 模板系统，新增模板类型 `password_reset`、`email_verify`，
   管理员可在设置里编辑、可重置为默认。链接/OTP 两种形态各自的模板变量。

## 2. 六块功能

### ① 验证码（beta.1，✅ 已实现）
- 多 provider：`image`（base64Captcha 自托管，CN 安全默认，零外部调用）/ `turnstile` / `recaptcha` / `hcaptcha`。
- `CaptchaVerifier` 接口 + 4 实现（image 有状态：TTL challenge store + `GET /auth/captcha` 发图；
  三家 token provider 无状态：前端 widget 取 token、后端拿 secret 走各自 siteverify）。
- 触发可配：`always` / `after_failures`（失败数按 `LockoutWindowMinutes` 窗口计，达 `CaptchaFailThreshold`）。
- 失败响应体带 `captcha_required:true`，前端据此揭示验证码；`always` 模式 `Methods()` 直接告知。

### ② 账号锁定（beta.1）
- 复用 `auth_events` 失败日志，按 `LockoutScope`（`ip` / `ip_upn`）在窗口内计数；
  达 `LockoutThreshold` 则锁 `LockoutDurationMinutes`，独立于、严于 per-IP 限流。
- `ip_upn` 为推荐默认：知道用户名也无法靠异地刷失败把受害者锁死。
- 新增精确计数：`AuthEventRepo.RecentAuthFailures(ctx, ip, upn, since) → (count, lastAt)`（现 filter 只有模糊 Search，不够安全用）。

### ③ 邮箱找回密码（= 发布序列 beta.3，✅ 已实现；通知中心插在 beta.2）
- 仅对「有 email + 有本地密码」的用户。开关 `password_recovery_enabled` + 需 SMTP 已配。投递 `link`/`otp` 可配（默认 link）。
- 实现：`auth_tokens` 表 + `AuthTokenRepo`（SHA-256 哈希、原子单次消费）；`internal/service/recovery`（零枚举、先验密码强度再消费令牌、链接基址优先 SubBaseURL 防投毒）；mailer `SendPasswordReset` + `password_reset` 模板；前端找回/重置页。复用 `user.Service.SetPassword`（bcrypt + TokenVersion bump 踢会话）。
- `POST /auth/forgot-password {ident}` → 无论存在与否都回 200（不泄露账号枚举）；存在则发邮件（链接或 OTP，按配置）。
- `POST /auth/reset-password {token|code+ident, new_password}` → 校验 `auth_tokens` → 重置 bcrypt + 失效 token +
  踢现有会话（refresh 版本号 +1，可选）。挂验证码+限流。

### ④ 自助注册（= 发布序列 beta.4，✅ 已实现）
- 开关 `registration_enabled`。`POST /auth/register {email,password,display_name}` —— **邮箱即 UPN**（登录名，唯一索引兼作去重；Group 无配额故配额/到期由注册设置提供）。角色固定 `user`、加入默认组。
- 门槛：**默认强制邮箱验证**（存储层反向 `allow_unverified` 使安全默认=要求验证）+ **邮箱域名白名单**（精确匹配）+ **默认组 + 默认配额/有效期**。不做邀请码。
- 开验证：建禁用账号（`DisabledPendingEmailVerify`，各登录闸门自动拦、不自助豁免）+ **不开通 3X-UI**；发 `email_verify` 邮件（复用 `auth_tokens` + OTP 5 次上限；link/OTP 可配）。`POST /auth/verify-email` 消费令牌 → `user.ActivateAfterVerification`（启用+此时才开通客户端；仅 pending 态生效防旧令牌复活停用号）。关验证则直接建+开通。
- 实现：`internal/service/registration`（TDD）；链接 URL 只用 `SubBaseURL` 防投毒；两端点挂 loginLimiter。
- **SSO 劫持加固**：自助注册行加 `User.SelfRegistered` 标记，`EnsureSSO` 首次 link 排除之（未验证 scrub、已激活返 `ErrSSOAccountConflict`）——杜绝抢注受害者邮箱后被 SSO 静默 rebind 接管/卡死（23-agent 审查 HIGH）。重复邮箱返 409（标准注册 UX，验证闸门防劫持）。

### ⑤ Passkey / WebAuthn（beta.6，✅ 已实现）
- 库 `go-webauthn/webauthn`（后端）+ `@simplewebauthn/browser`（前端）。开关 `passkey_enabled`（绑定）+ `passkey_passwordless`（无密码 discoverable 登录）。仅本地密码账号（`HasLocalPassword`，排除 SSO）。
- 注册（已登录，个人资料页）：`POST /user/me/passkeys/{begin,finish}` → 存 `webauthn_credentials`。**凭据整条 `webauthn.Credential` JSON 落库**（库推荐形态，保留 Flags/Authenticator/CloneWarning）；`credential_id`（base64url）**全局唯一**、`sign_count` 反范式化出来供防回滚门；另 `user_id/name/created_at/last_used_at`。列出/重命名/删除均用户作用域。
- 登录：`POST /auth/passkey/{begin,finish}`（usernameless discoverable：begin 无 allow-list、finish 经 `DiscoverableUserHandler` 据 credential 反解用户）→ 验 assertion → 复用密码登录同款 loginResponse（**停用闸 + 非管理员 `DisallowUserLocalLogin` 闸 + 2FA 闸**；已开 TOTP 则回 `2fa_required`）。
- **RP ID/Origin 严格取自 `SubBaseURL`**（裸主机名 + scheme://host），**空则硬失败不回退请求 Host**（防 RP-ID 投毒）。begin↔finish 的 challenge 用内存**单次** session store（take 即删、5min TTL，绝不放进 JWT）。
- **防克隆**：`go-webauthn` 对 sign-count 回退置 `CloneWarning`（不报错、保留旧计数、豁免全零）；`finalizeAssertion` 见 `CloneWarning` 即拒登。仓储 `UpdateAfterLogin` 的 `sign_count<=` 门作单调写、不致登录失败（避并发误杀）。
- **未做（留后续可选）**：passkey 当“第二因子”模式（现仅免密主登录 + 个人页绑定）；attestation 校验/MDS。

### ⑥ 2FA（TOTP）（beta.5，✅ 已实现）
- 库 `pquerna/otp`（默认 ±1 步容差）。开关 `totp_enabled`（关只阻止新开启，不剥夺已启用账号）。仅本地密码账号（`Begin` 要求 `HasLocalPassword`）。
- 启用（已登录）：`POST /user/me/2fa/begin`（出 secret + otpauth URI，前端 `qrcode.react` 渲二维码，密钥**禁用态**存库）+ `/user/me/2fa/enable`（验码确认）→ 标记启用 + 生成 10 个一次性**恢复码**（仅显示一次，SHA-256 哈希存）。`/user/me/2fa/disable` 需当前 TOTP 或恢复码。
- 登录：密码过后若用户已启用 2FA → 返回 `{status:"2fa_required", pending_token}`（5 分钟 `2fa_pending` JWT）→ `POST /auth/2fa/verify {pending_token, code}` → 校验后发 JWT（共用登录限流）。**verify 完成前照 Refresh 路径重核 `Enabled`/`TokenVersion`**，防 pending 窗内吊销的账号还能换会话。
- 管理员可重置某用户的 2FA（`POST /admin/users/:id/reset-2fa`，operator 越权防护）。用户存 `totp_secret(加密)/totp_enabled/recovery_codes(哈希)`，均经 `pollOwnedColumns` 保护。
- **恢复码单次使用并发安全**：消费走 `ConsumeRecoveryCode` 的 compare-and-swap（`WHERE recovery_codes=<prev>` + `RowsAffected==1`），杜绝两个并发 verify 携同一码双花。
- **未做**：`totp_enforced_for_admins`（强制管理员启用）与 TOTP 码 ~90s 窗内防重放——均留作后续可选加固。

## 8. 与现有体系的关系
全是本地认证概念。SSO（SAML/OIDC）用户无本地密码 → 不受找回/注册/2FA/passkey 影响；
严守「身份零隐式 fallback」。SSO 侧 MFA 由 IdP 负责，PSP **不做** SSO 二次验证（已与维护者确认）。
所有新公开端点统一挂在 beta.1 的「验证码 + 限流」闸门后。
